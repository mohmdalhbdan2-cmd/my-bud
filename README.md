# ميزانيتي 💰

تطبيق ويب كامل لإدارة الميزانية الشخصية والعائلية — تحويل لفكرة ملف الإكسل الأصلي إلى تطبيق تفاعلي بقاعدة بيانات حقيقية، يشتغل على أي جهاز (وممكن تثبيته على آيفون كأنه تطبيق حقيقي).

## المحتويات

```
ميزانيتي-app/
├── src/
│   ├── App.jsx              ← كل واجهة التطبيق (Dashboard, الشهر, السنة, المدخرات, الديون, الأهداف, الإعدادات)
│   ├── main.jsx              ← نقطة الدخول
│   ├── index.css             ← Tailwind + خطوط عربية
│   └── lib/
│       └── supabase.js       ← كل دوال الاتصال بقاعدة البيانات (CRUD كامل)
├── supabase/
│   └── schema.sql            ← السكيما الكاملة لقاعدة البيانات (نفّذها مرة واحدة في Supabase)
├── public/
│   └── manifest.json         ← لدعم "إضافة للشاشة الرئيسية" في آيفون
├── package.json
├── vite.config.js
├── tailwind.config.js
├── .env.example               ← انسخه إلى .env.local وعبّي مفاتيح Supabase
└── index.html
```

## الخطوة 1: إنشاء مشروع Supabase (5 دقائق)

1. روح [supabase.com](https://supabase.com) وسجّل حساب مجاني.
2. اضغط **New Project**، اختر اسم وكلمة مرور لقاعدة البيانات واحفظها.
3. بعد ما يجهز المشروع (دقيقة أو دقيقتين)، روح **SQL Editor** من القائمة الجانبية.
4. افتح ملف `supabase/schema.sql` من هذا المشروع، انسخ **كل محتواه**، الصقه في SQL Editor، واضغط **Run**.
   - هذا ينشئ كل الجداول (11 جدول)، الصلاحيات الأمنية (RLS)، والفهارس تلقائياً.
5. روح **Settings → API**، وانسخ:
   - `Project URL` → هذا هو `VITE_SUPABASE_URL`
   - `anon public key` → هذا هو `VITE_SUPABASE_ANON_KEY`

## الخطوة 2: تفعيل تسجيل الدخول بالإيميل

1. في Supabase، روح **Authentication → Providers**.
2. تأكد إن **Email** مفعّل (مفعّل افتراضياً).
3. لو تبي تسجيل دخول بدون تأكيد إيميل أثناء التجربة: **Authentication → Settings**، عطّل "Confirm email".

## الخطوة 3: تشغيل المشروع محلياً

```bash
cd ميزانيتي-app
npm install
cp .env.example .env.local
# افتح .env.local واملأ القيم من الخطوة 1
npm run dev
```

افتح المتصفح على `http://localhost:5173`.

---

## الخطوة 4: رفع المشروع على الإنترنت (Vercel — مجاني)

1. ارفع المشروع على GitHub (أنشئ repo جديد وادفع الكود).
2. روح [vercel.com](https://vercel.com)، سجّل دخول بـ GitHub.
3. **New Project** → اختر الـ repo.
4. في **Environment Variables**، أضف نفس القيمتين من `.env.local`:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
5. اضغط **Deploy**. بعد دقيقة بيعطيك رابط مثل `mizaniyati.vercel.app`.

---

## الخطوة 5: تثبيته على آيفون كتطبيق حقيقي

1. افتح رابط الموقع (`mizaniyati.vercel.app`) من **Safari** في الآيفون (لازم Safari تحديداً، مو كروم).
2. اضغط زر **المشاركة** (المربع وفيه سهم لأعلى) في أسفل الشاشة.
3. مرّر لتحت واختر **إضافة إلى الشاشة الرئيسية** (Add to Home Screen).
4. بيظهر أيقونة التطبيق على شاشتك الرئيسية، وبيفتح بدون شريط عنوان المتصفح — يعني تجربة تطبيق حقيقي.

هذا يُسمى **PWA (Progressive Web App)**. ملف `manifest.json` الموجود في `public/` هو المسؤول عن هذا.

## مميزات التطبيق

### 1. الصفحة الرئيسية (Dashboard)
- ملخص الشهر الحالي تلقائياً (بدون ما تحدد شيء).
- تنبيه أحمر إذا الرصيد المتبقي صار سالب.
- تنبيه أصفر إذا ما حققت هدف الادخار.
- 4 بطاقات إحصائية: الدخل، المصاريف، المدخرات، الرصيد المتبقي.
- Pie Chart لتصنيف المصاريف المتغيرة (13 تصنيف: بنزين، مطعم، بقالة، فواتير، ملابس، صحة، ترفيه، نقل، تعليم، هدايا، حلاقة، طلعة شباب، أخرى).
- رسم بياني (Area Chart) لاتجاه الدخل والرصيد عبر السنة.
- تنبيهات بالديون القريبة من الاستحقاق.

### 2. صفحة الشهر
- 4 جداول قابلة للتعديل: الدخل، الالتزامات الثابتة، المصاريف المتغيرة (مع تصنيف)، المدخرات.
- إضافة/حذف بنود بسهولة (زر "+" لكل جدول).
- تنبيه لوني أحمر تلقائي إذا "الفعلي" تجاوز "المتوقع" في أي بند مصروف.
- **الرصيد المرحّل من الشهر السابق منفصل تماماً عن "الدخل"** — ما يدخل في حسابات النسب المئوية، فقط في حساب الرصيد المتاح للصرف (هذا تصحيح للخطأ الذي كان موجوداً في نسخة الإكسل).
- شريط منزلق لتحديد هدف الادخار (% من الدخل) مع مؤشر "محقق / لم يحقق".
- ملخص نهائي بالنسب المئوية لكل بند.

### 3. صفحة السنة
- إحصائيات سنوية كاملة (دخل، مصاريف، مدخرات، رصيد نهائي).
- Bar Chart مقارنة شهرية.
- جدول تفصيلي لكل الـ 12 شهر، بالضغط على أي شهر ينقلك لتفاصيله مباشرة.
- إمكانية التنقل بين السنين (سهم يمين/يسار) — **التطبيق يدعم عدد غير محدود من السنين**، كل سنة ترث الرصيد المرحّل من نهاية السنة السابقة تلقائياً.

### 4. صفحة المدخرات والمحفظة
- رسم بياني تراكمي (Area Chart) يوضح نمو مدخراتك عبر كل السنين مجتمعة.
- جدول مدخرات كل سنة على حدة.
- جدول "مدخرات سابقة" لتسجيل أي مبلغ كان عندك قبل ما تبدأ تستخدم التطبيق، مع تصنيف مكانها وعائدها.

### 5. صفحة الديون والقروض
**قروض رسمية (بالتقسيط):**
- تكتب اسم الدين، المبلغ الكلي، تاريخ البداية.
- تكتب إما **عدد الأشهر** أو **المبلغ الشهري** فقط — والثاني يُحسب تلقائياً (حاسبة تبادلية).
- تاريخ الانتهاء يُحسب تلقائياً.
- شريط تقدّم يوضح كم شهر دفعت من الإجمالي.
- **الدفعة الشهرية تظهر تلقائياً فقط في الأشهر التي يكون القرض نشطاً فيها فعلياً** (بين تاريخ البداية والنهاية)، وتُحسب ضمن "الالتزامات الثابتة" لذلك الشهر تحديداً — مو بالتساوي على كل الأشهر.

**ديون شخصية (لأشخاص):**
- اسم الشخص، المبلغ، تاريخ الاستلام، تاريخ السداد المتوقع (اختياري).
- عداد أيام متبقية للسداد، يتحدّث تلقائياً حسب تاريخ اليوم الفعلي.
- زر "تم السداد" يحدّث الحالة فوراً.
- إجمالي الديون الشخصية غير المسددة.

### 6. صفحة الأهداف والمشتريات
- قائمة أمنيات/احتياجات غير مرتبطة بشهر معين.
- تصنيف (احتياج/حاجة/رغبة)، أولوية، حالة (لم يُشترى/قيد التوفير/تم الشراء).
- إجمالي تكلفة الأهداف غير المحققة بعد.

### 7. الإعدادات
- تحديد شهر وسنة بداية الاستخدام الفعلي.
- الرصيد الحالي عند البداية.
- ربط Supabase (URL + Key).

## المشاركة العائلية / الفريق

السكيما مبنية لدعم المشاركة من البداية عبر جدولي `households` و `household_members`:

1. أي مستخدم يقدر ينشئ household (عائلة/فريق) عبر `householdApi.create()`.
2. يدعو أعضاء آخرين عبر `householdApi.inviteMember()`.
3. أي بيانات (دخل، مصاريف، ديون...) تُسجَّل بـ `household_id`، تصير مرئية لكل الأعضاء.
4. الصلاحيات الأمنية (RLS) في `schema.sql` مفعّلة بحيث محد يشوف بيانات عائلة ثانية.

*(الواجهة الحالية في `App.jsx` تستخدم mock data محلي مبسّط — لتفعيل المشاركة الفعلية، اربط `householdApi` بشاشة "الإعدادات" وأضف اختيار household عند كل عملية كتابة).*

## الخطوة التالية: ربط الواجهة بقاعدة البيانات الفعلية

حالياً `App.jsx` يستخدم `useState` محلي (mock data) عشان تقدر تجرب التطبيق فوراً بدون أي إعداد. لتفعيل الحفظ الحقيقي في Supabase:

1. استورد الدوال من `src/lib/supabase.js` بدل الـ mock function الموجودة فوق الملف.
2. في كل مكان فيه `setYears`, `setLoans`, `setGoals`, إلخ — استبدلها بنداء API مثل:
   ```js
   await incomeApi.create({ user_id, year, month, name, estimated, actual });
   ```
3. أضف `useEffect` لجلب البيانات عند تحميل كل صفحة:
   ```js
   useEffect(() => {
     incomeApi.list(userId, currentYear, currentMonth).then(setIncomeEntries);
   }, [currentYear, currentMonth]);
   ```
4. أضف شاشة تسجيل دخول (`auth.signIn` / `auth.signUp`) قبل عرض التطبيق.

كل دالة جاهزة ومُختبرة منطقياً في `supabase.js` — فقط تحتاج ربطها بأحداث الواجهة (onClick, onChange).

## ملاحظات تقنية مهمة

- **حساب "الدخل الفعلي" منفصل عن "الرصيد المرحّل"**: هذا تصحيح جوهري عن نسخة الإكسل الأصلية — الرصيد المرحّل لا يُحتسب ضمن نسب الدخل المئوية لأنه ليس دخلاً حقيقياً لذلك الشهر.
- **القروض مرتبطة بالتاريخ الفعلي**: أي قرض يظهر فقط في الأشهر الواقعة بين `start_date` و `end_date` المحسوبين، باستخدام منطق تقاطع نطاقات التاريخ (date range overlap)، تماماً كما في صيغة `SUMPRODUCT` المستخدمة بملف الإكسل الأصلي.
- **التطبيق متعدد السنين بطبيعته**: كل سنة جديدة تُحسب تلقائياً بالاعتماد على نهاية السنة السابقة (`computedYearData`)، فلا حاجة لإنشاء "ملف جديد" كل سنة كما في الإكسل.
- **الوضع الليلي (Dark Mode)** مدمج بالكامل عبر Tailwind `dark:` classes.
- **التصميم متجاوب بالكامل (Responsive)**: الشريط الجانبي يتحول لقائمة منسدلة على الجوال، والجداول تتمرر أفقياً عند الحاجة.
